📖 What Is Machine Learning?

Machine learning is a branch of artificial intelligence that gives computer systems the ability to learn and improve from experience without being explicitly programmed. Rather than writing rules by hand, we provide a model with data and let it discover the rules on its own.

Tom Mitchell's classical definition: "A computer program is said to learn from experience E with respect to task T and performance measure P, if its performance on T as measured by P improves with experience E."

ML has become the dominant approach to AI because modern data abundance and computing power make it practical to train models on millions of examples — reaching human-level or superhuman performance on many narrow tasks.

🔀 The Three Learning Paradigms

1. Supervised Learning

The most common paradigm. The training set consists of labeled examples — pairs of (input X, correct output y). The model learns a mapping f(X) → y by minimizing prediction error on the training data, then generalizes to unseen examples.

  • Classification — Predict a discrete class label. Example: Is this email spam or not-spam? Is this tumor malignant or benign? What digit (0–9) does this image show?
  • Regression — Predict a continuous numeric value. Example: What will this house sell for? What will the S&P 500 close at tomorrow?

2. Unsupervised Learning

No labels are provided. The model must discover hidden structure in the data on its own — finding natural groupings, compressing representations, or identifying anomalies.

  • Clustering — Group similar data points. Algorithms: K-Means, DBSCAN, hierarchical clustering. Used in customer segmentation and document grouping.
  • Dimensionality Reduction — Compress high-dimensional data while preserving structure. Algorithms: PCA, t-SNE, UMAP. Used for visualization and feature extraction.
  • Anomaly Detection — Identify unusual patterns that deviate from the norm. Used in fraud detection and network intrusion detection.
  • Generative Models — Learn the data distribution to generate new samples. Examples: GANs, VAEs, diffusion models (Stable Diffusion).

3. Reinforcement Learning

An agent interacts with an environment, taking actions that yield rewards or penalties. Over many episodes, the agent learns a policy — a strategy for choosing actions that maximizes long-term cumulative reward.

  • Powered AlphaGo's defeat of the world Go champion in 2016
  • Trains ChatGPT to follow human instructions (RLHF)
  • Optimizes robotic manipulation tasks
  • Used in algorithmic trading for portfolio management

⚙️ The Machine Learning Pipeline

Building a production ML system is a multi-stage engineering process. Each stage is critical — a failure at any point undermines the entire system.

1. Problem Definition

Clearly define the task: What is X? What is y? What metric defines success? What decisions will the model's output drive?

2. Data Collection & Exploration

Gather data from databases, APIs, scrapers, or sensors. Perform Exploratory Data Analysis (EDA): examine distributions, correlations, missing values, and class imbalance.

3. Data Preprocessing

  • Handling missing values — Impute with mean/median, or drop rows/columns.
  • Normalization / standardization — Scale features so no single variable dominates (Z-score, Min-Max).
  • Encoding categoricals — One-hot encoding, ordinal encoding, or learned embeddings.
  • Train/validation/test split — Typical: 70% train, 15% validation, 15% test. Never touch the test set until final evaluation.

4. Feature Engineering

The art of transforming raw data into features that expose patterns to the model. Often the single biggest lever for improving performance. Techniques include: polynomial features, interaction terms, log transforms, rolling statistics, and domain-specific derivations.

5. Model Selection & Training

Choose an algorithm based on data size, dimensionality, interpretability requirements, and latency constraints. Train on the training set, tune hyperparameters on the validation set.

6. Evaluation & Hyperparameter Tuning

Grid search or Bayesian optimization over hyperparameter space. Use cross-validation to get reliable performance estimates. Evaluate on the held-out test set exactly once.

7. Deployment & Monitoring

Serve the model via REST API, batch job, or edge device. Monitor for data drift (input distribution changes) and concept drift (relationship between X and y changes over time). Retrain regularly.

🧮 Core Algorithms

Linear & Logistic Regression

The simplest supervised models. Linear regression fits a straight line (hyperplane) through data to predict continuous values. Logistic regression applies a sigmoid to predict binary probabilities. Both are fast, interpretable, and serve as strong baselines.

Linear: ŷ = w₀ + w₁x₁ + w₂x₂ + … + wₙxₙ
Logistic: P(y=1) = σ(w·x + b) = 1 / (1 + e^−(w·x+b))

Decision Trees & Random Forests

A decision tree partitions the feature space with a sequence of binary splits, creating a flowchart-like structure. Easy to interpret, but prone to overfitting.

A Random Forest ensembles hundreds of independently-trained trees, each on a random subset of data and features. Averaging their predictions dramatically reduces variance and overfitting — making Random Forests one of the most reliable general-purpose algorithms.

Gradient Boosting (XGBoost, LightGBM)

Builds an ensemble of weak learners sequentially — each tree corrects the errors of the previous. The dominant algorithm in tabular data competitions. XGBoost and LightGBM add regularization, parallelism, and missing-value handling for exceptional performance.

Support Vector Machines (SVM)

Finds the maximum-margin hyperplane separating classes in feature space. The kernel trick allows SVMs to learn non-linear boundaries by implicitly mapping data to higher-dimensional spaces. Effective for high-dimensional, small-sample problems.

Neural Networks

The most flexible algorithm — see Module 1 for a full treatment. Neural networks dominate unstructured data (images, text, audio) but require more data and compute than classical ML algorithms.

📏 Model Evaluation

Classification Metrics

  • Accuracy — Fraction of correct predictions. Misleading on imbalanced datasets.
  • Precision — Of predicted positives, fraction that are actually positive. High precision = few false alarms.
  • Recall (Sensitivity) — Of actual positives, fraction detected. High recall = few missed cases.
  • F1 Score — Harmonic mean of precision and recall. Balanced single metric when both matter.
  • ROC-AUC — Area under the Receiver Operating Characteristic curve. Measures discrimination ability across all thresholds; 1.0 = perfect, 0.5 = random.
  • Confusion Matrix — 2×2 table of true/false positives and negatives. Essential for understanding error types.

Regression Metrics

  • MAE (Mean Absolute Error) — Average absolute deviation. Robust to outliers.
  • RMSE (Root Mean Squared Error) — Penalizes large errors more heavily. Standard in practice.
  • R² (R-squared) — Fraction of variance explained by the model. 1.0 = perfect fit, 0 = no better than predicting the mean.

Cross-Validation

k-Fold cross-validation splits data into k folds, trains on k−1 folds, and evaluates on the remaining fold — rotating through all combinations. The average score gives a low-variance estimate of true out-of-sample performance.
