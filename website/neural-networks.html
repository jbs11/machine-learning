📖 What Are Neural Networks?

A neural network is a computational model loosely inspired by the structure of the human brain. Just as the brain consists of billions of neurons connected by synapses, an artificial neural network consists of nodes (artificial neurons) connected by weighted edges.

The key insight is that neural networks can learn any function from input to output, given enough data and capacity — a property known as universal approximation. This makes them extraordinarily flexible: the same architecture that recognizes faces can also translate languages, predict stock prices, or generate music.

Neural networks are the engine behind deep learning — the sub-field of machine learning that uses networks with many layers to extract hierarchically complex features from raw data.

⚡ How a Single Neuron Works

The fundamental building block is the perceptron — an artificial neuron that performs a weighted sum of its inputs and passes the result through a non-linear activation function.

The Computation

Given inputs x₁, x₂, …, xₙ, each with a corresponding learned weight w, the neuron computes:

z = (x₁·w₁) + (x₂·w₂) + … + (xₙ·wₙ) + b
output = f(z) ← activation function applied to weighted sum

The bias term b shifts the activation threshold, allowing the neuron to activate even when all inputs are zero. It gives the model an extra degree of freedom.

Activation Functions

The activation function introduces non-linearity, which is what allows stacked layers to learn complex patterns. Without it, any deep network would collapse to a single linear transformation — no more powerful than logistic regression.

ReLU

max(0, z) — Fast, avoids vanishing gradient. Default choice for hidden layers.

Sigmoid

1/(1+e⁻ᶻ) — Outputs 0–1. Used in binary output layers for probabilities.

Tanh

Outputs −1 to +1. Zero-centered; often preferred over sigmoid in hidden layers.

Softmax

Converts logits to probabilities that sum to 1. Used in multi-class outputs.

Leaky ReLU

max(0.01z, z) — Fixes the dying ReLU problem by allowing small negative gradients.

GELU

Smooth approximation of ReLU used in Transformers (BERT, GPT).

🏗 Network Architecture

Individual neurons are arranged into layers. Each layer transforms its input into a new representation, and deeper networks extract increasingly abstract features.

Layer Types

  • Input Layer — Receives raw data. Size equals the number of input features (pixels, sensor readings, word embeddings, etc.).
  • Hidden Layers — One or more intermediate layers where feature extraction happens. Each layer learns to detect progressively more complex patterns.
  • Output Layer — Produces the final prediction. Size depends on the task: 1 node for regression, N nodes for N-class classification.

Depth vs. Width

Depth (number of layers) determines the level of abstraction a network can represent. Early layers detect low-level features (edges, frequencies); deeper layers detect complex composites (faces, semantics).

Width (neurons per layer) controls representational capacity within each level of abstraction. Wider layers can memorize more patterns but also overfit more easily.

Fully Connected vs. Specialized Architectures

  • Multi-Layer Perceptron (MLP) — Every neuron connects to every neuron in the next layer. General purpose; good for tabular data.
  • Convolutional Neural Network (CNN) — Shares weights across spatial positions via filters. Ideal for images and sequences.
  • Recurrent Neural Network (RNN/LSTM) — Processes sequences with temporal memory. Used for time series and NLP.
  • Transformer — Uses self-attention to model long-range dependencies. Powers GPT, BERT, Stable Diffusion.

➡️ The Forward Pass

During inference, data flows from left to right through the network — this is called the forward pass. At each layer, a matrix multiplication is performed (computing all weighted sums in parallel) followed by the activation function.

Layer output = activation( W · input + b )

Modern frameworks like PyTorch and TensorFlow use GPU-accelerated matrix operations to make this extremely fast. A forward pass through a 50-layer network on a modern GPU takes milliseconds.

The forward pass is also used during training — predictions are computed first, then compared to the true labels to compute a loss, which drives backpropagation.

🎯 Training: Backpropagation & Gradient Descent

Training is the process of adjusting weights so the network's predictions become more accurate. It combines two algorithms: loss computation and backpropagation.

Step 1 — Compute Loss

After a forward pass, the loss function measures prediction error. Common choices:

  • Mean Squared Error (MSE) — For regression: average of squared differences between predicted and actual values.
  • Cross-Entropy Loss — For classification: measures how wrong the predicted probability distribution is vs. the true label.

Step 2 — Backpropagation

Backpropagation applies the chain rule of calculus to efficiently compute the gradient of the loss with respect to every weight in the network — working from the output layer backward to the input layer.

∂Loss/∂w = ∂Loss/∂output · ∂output/∂z · ∂z/∂w

Step 3 — Update Weights (Gradient Descent)

Each weight is nudged in the direction that reduces the loss:

w ← w − η · (∂Loss/∂w)

where η (eta) is the learning rate — a hyperparameter controlling step size.

Optimizers

  • SGD (Stochastic Gradient Descent) — Updates after each mini-batch. Simple and effective with momentum.
  • Adam — Adapts the learning rate per parameter using moment estimates. The default choice for most tasks.
  • RMSProp — Divides gradients by a running average of magnitudes. Good for RNNs.

🛡 Overfitting & Regularization

A network with enough capacity can memorize its training data perfectly — but then fails on new data. This is called overfitting. Regularization techniques prevent it:

  • Dropout — Randomly sets a fraction of neurons to zero during training, forcing the network to learn redundant representations.
  • Batch Normalization — Normalizes layer inputs across each mini-batch, stabilizing and accelerating training.
  • Weight Decay (L2) — Adds a penalty proportional to weight magnitude, shrinking large weights toward zero.
  • Data Augmentation — Artificially expands training data by transforming examples (flips, rotations, noise), improving generalization.
  • Early Stopping — Halts training when validation loss stops improving, preventing the model from overfitting further.
